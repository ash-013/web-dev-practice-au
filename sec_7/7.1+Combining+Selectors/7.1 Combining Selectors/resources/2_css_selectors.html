1. Basic Selectors

1.1 Universal Selector (*)

Selects ALL elements on the page.

/* Selects every single element */ * { margin: 0; padding: 0; box-sizing: border-box; } /* With namespace */ ns|* { } /* All elements in namespace ns */ *|* { } /* All elements in all namespaces */ |* { } /* All elements without namespace */
Use case: CSS resets, applying box-sizing to everything.

1.2 Type Selector (Element Selector)

Selects all elements of a given type.

/* Selects all <p> elements */ p { line-height: 1.6; } /* Selects all <div> elements */ div { display: block; } /* Selects all <h1> elements */ h1 { font-size: 2em; }

This is a paragraph affected by the p selector.

This is a div affected by the div selector.

1.3 Class Selector (.)

Selects all elements with a specific class attribute.

/* Selects all elements with class="button" */ .button { padding: 10px 20px; background: blue; color: white; } /* Multiple classes - selects elements with BOTH classes */ .button.primary { background: darkblue; } /* Can be used multiple times */ .nav .item .link { color: blue; }

1.4 ID Selector (#)

Selects a single element with a specific ID (IDs should be unique).

/* Selects element with id="header" */ #header { background: navy; color: white; } /* Avoid using IDs for styling - use classes instead */ /* IDs are best for JavaScript hooks */ /* Can be combined */ div#main { max-width: 1200px; }
‚ö†Ô∏è Warning: IDs have very high specificity (100). Prefer classes for styling to keep specificity manageable.

2. Attribute Selectors

Select elements based on their attributes and attribute values.

2.1 Presence Selector [attr]

/* Selects elements that have the attribute */ [disabled] { opacity: 0.5; cursor: not-allowed; } [required] { border-left: 3px solid red; } [data-tooltip] { position: relative; }

2.2 Exact Match [attr="value"]

/* Exact match */ [type="text"] { border: 1px solid gray; } [type="email"] { border: 1px solid blue; } [rel="nofollow"] { color: gray; }

2.3 Contains Word [attr~="value"]

/* Matches if attribute contains the word (space-separated) */ [class~="btn"] { padding: 10px; } /* Matches: class="btn primary" */ /* Matches: class="large btn" */ /* Doesn't match: class="button" or "btn-primary" */ [title~="flower"] { border: 2px solid green; } /* Matches: title="red flower" or "flower garden" */

2.4 Starts With [attr^="value"]

/* Selects elements where attribute starts with value */ [class^="btn-"] { border-radius: 5px; } /* Matches: class="btn-primary", "btn-secondary", "btn-large" */ [href^="https"] { color: green; } /* Matches secure links */ [src^="/images"] { border: 2px solid blue; } /* Matches images from /images folder */

2.5 Ends With [attr$="value"]

/* Selects elements where attribute ends with value */ [href$=".pdf"] { background: url('pdf-icon.png') no-repeat left; padding-left: 20px; } [src$=".jpg"], [src$=".jpeg"], [src$=".png"] { border: 3px solid #ddd; } [class$="-disabled"] { opacity: 0.5; } /* Matches: "btn-disabled", "input-disabled" */
üí° Tip: Great for styling links to different file types (PDFs, DOCs, etc.).

2.6 Contains Substring [attr*="value"]

/* Matches if attribute contains substring anywhere */ [class*="col-"] { float: left; } /* Matches: "col-6", "large-col-4", "col-md-8" */ [href*="youtube"] { color: red; } /* Matches any YouTube link */ [title*="important"] { font-weight: bold; }

2.7 Language Prefix [attr|="value"]

/* Matches exact value or value followed by hyphen */ [lang|="en"] { color: blue; } /* Matches: lang="en", lang="en-US", lang="en-GB" */ /* Doesn't match: lang="eng" */ [data-country|="us"] { background: url('us-flag.png'); }

2.8 Case-Insensitive [attr="value" i]

/* Case-insensitive matching (add 'i' flag) */ [type="EMAIL" i] { border: 1px solid blue; } /* Matches: type="email", type="EMAIL", type="Email" */ [href*="PDF" i] { font-weight: bold; } /* Matches .pdf, .PDF, .Pdf */

2.9 Complete Attribute Selector Examples

/* HTML */ <a href="https://example.com/doc.pdf" class="btn-primary">Download</a> <input type="email" required> <div data-state="active" data-priority="high"></div> /* CSS */ /* External links */ a[href^="http"]:not([href*="yourdomain.com"]) { color: purple; } /* Download links with icons */ a[href$=".pdf"]::before { content: "üìÑ "; } a[href$=".zip"]::before { content: "üì¶ "; } /* Style by data attributes */ [data-state="active"] { background: green; } [data-priority="high"] { border: 3px solid red; } /* Email inputs */ input[type="email"][required] { border-left: 3px solid orange; }

3. Combinator Selectors

Combinators explain the relationship between selectors.

3.1 Descendant Selector (space)

/* Selects all <p> inside <div> (any level deep) */ div p { color: blue; } /* Selects all spans inside elements with class="container" */ .container span { font-weight: bold; } /* Multiple levels */ article div p span { color: red; }

3.2 Child Selector (>)

/* Selects only DIRECT children */ div > p { color: green; } /* Only immediate children with class="item" */ .menu > .item { display: inline-block; } /* Multiple levels */ ul > li > a { text-decoration: none; }

4. Pseudo-class Selectors

Pseudo-classes select elements based on their state or position.

4.1 Dynamic Pseudo-classes (User Actions)

/* Link states (in this order!) */ a:link { color: blue; /* Unvisited links */ } a:visited { color: purple; /* Visited links */ } a:hover { color: red; /* Mouse over */ } a:active { color: orange; /* Being clicked */ } /* Focus state */ input:focus { outline: 2px solid blue; background: lightyellow; } button:focus { box-shadow: 0 0 0 3px rgba(0,123,255,0.3); } /* Remember: LVHA order (Link, Visited, Hover, Active) */
Hover over this link

4.2 Structural Pseudo-classes (Position-based)

:first-child and :last-child

/* First child of parent */ li:first-child { font-weight: bold; } /* Last child of parent */ li:last-child { border-bottom: none; } /* First paragraph in div */ div p:first-child { margin-top: 0; }

:first-of-type and :last-of-type

/* First <p> among siblings */ p:first-of-type { font-size: 1.2em; } /* Last <img> in container */ img:last-of-type { margin-bottom: 0; } /* Difference from :first-child */ div p:first-of-type { } /* First p, even if div has other elements first */ div p:first-child { } /* p that is first child of div */

:nth-child() - Powerful Pattern Matching

/* Specific position */ li:nth-child(3) { color: red; /* 3rd child */ } /* Odd rows */ tr:nth-child(odd) { background: #f9f9f9; } /* Even rows */ tr:nth-child(even) { background: white; } /* Every 3rd element */ li:nth-child(3n) { color: blue; /* 3, 6, 9, 12... */ } /* Every 3rd element starting from 2nd */ li:nth-child(3n+2) { color: green; /* 2, 5, 8, 11... */ } /* First 5 elements */ li:nth-child(-n+5) { font-weight: bold; /* 1, 2, 3, 4, 5 */ } /* All except first 3 */ li:nth-child(n+4) { opacity: 0.7; /* 4, 5, 6, 7... */ }

  • Item 4 (even)
  • Item 5 (odd)

:nth-of-type()

/* Every odd paragraph */ p:nth-of-type(odd) { background: #f0f0f0; } /* First 3 images */ img:nth-of-type(-n+3) { border: 3px solid blue; } /* Every 4th div */ div:nth-of-type(4n) { margin-bottom: 40px; }

:only-child and :only-of-type

/* Element that is only child of parent */ p:only-child { margin: 0; } /* Only <p> among siblings (other elements may exist) */ p:only-of-type { text-align: center; }

4.3 Negation Pseudo-class :not()

/* All inputs except submit buttons */ input:not([type="submit"]) { border: 1px solid gray; } /* All divs without class "special" */ div:not(.special) { background: white; } /* All paragraphs that aren't first child */ p:not(:first-child) { margin-top: 1em; } /* Multiple :not() */ button:not(.primary):not(.secondary) { background: gray; } /* Complex negation */ a:not([href*="external"]):not([target="_blank"]) { color: blue; }
üí° Tip: :not() is extremely powerful for excluding specific elements from rules.

4.4 Form Pseudo-classes

/* Checked checkboxes/radio buttons */ input:checked { outline: 2px solid green; } /* Disabled elements */ input:disabled, button:disabled { opacity: 0.5; cursor: not-allowed; } /* Enabled elements */ input:enabled { background: white; } /* Required fields */ input:required { border-left: 3px solid red; } /* Optional fields */ input:optional { border-left: 3px solid green; } /* Valid/Invalid fields */ input:valid { border-color: green; } input:invalid { border-color: red; } /* In range/Out of range (for number inputs) */ input:in-range { border-color: green; } input:out-of-range { border-color: red; } /* Read-only fields */ input:read-only { background: #f5f5f5; } /* Placeholder shown */ input:placeholder-shown { border-style: dashed; }

4.5 Other Useful Pseudo-classes

/* Empty elements */ div:empty { display: none; } /* Root element (html) */ :root { --main-color: blue; /* CSS variables */ } /* Target (element with ID matching URL hash) */ :target { background: yellow; animation: highlight 2s; } /* Language */ :lang(en) { quotes: '"' '"'; } :lang(fr) { quotes: '¬´' '¬ª'; }

CSS2: Single colon (:before, :after)

CSS3: Double colon (::before, ::after)

Best Practice: Use double colon (::) for pseudo-elements. Browsers support both for backward compatibility.

6. Advanced Selectors

6.1 :is() Pseudo-class (Matches Any)

/* Without :is() - repetitive */ h1 a, h2 a, h3 a, h4 a { color: blue; } /* With :is() - cleaner */ :is(h1, h2, h3, h4) a { color: blue; } /* Complex example */ :is(section, article, aside) :is(h1, h2, h3) { margin-top: 0; } /* Equivalent to: */ section h1, section h2, section h3, article h1, article h2, article h3, aside h1, aside h2, aside h3 { margin-top: 0; }
üí° Tip: :is() takes the specificity of its most specific argument.

6.2 :where() Pseudo-class (Zero Specificity)

/* :where() has 0 specificity */ :where(h1, h2, h3) { margin-top: 0; /* Specificity: 0,0,0,0 */ } /* This wins even though it's less specific structurally */ h2 { margin-top: 20px; /* Specificity: 0,0,0,1 - WINS! */ } /* Use case: default styles that are easy to override */ :where(.button) { padding: 10px; background: blue; } /* Easy to override */ .button { background: red; /* This wins */ }

6.3 :has() Pseudo-class (Parent Selector)

/* Style parent based on children! */ /* Div that contains a paragraph */ div:has(p) { background: lightyellow; } /* Form with invalid inputs */ form:has(input:invalid) { border: 2px solid red; } /* Article with images */ article:has(img) { display: grid; } /* Label followed by required input */ label:has(+ input:required) { font-weight: bold; } /* Card with no image */ .card:not(:has(img)) { padding: 20px; } /* Complex: section with h2 followed by paragraph */ section:has(h2 + p) { margin-bottom: 40px; }
‚ö†Ô∏è Note: :has() is a newer feature with limited browser support. Check compatibility before using.

6.4 Multiple Selectors (Grouping)

/* Apply same styles to multiple selectors */ h1, h2, h3, h4, h5, h6 { font-family: 'Georgia', serif; font-weight: bold; } /* Multiple complex selectors */ .nav a:hover, .menu a:hover, .footer a:hover { color: red; } /* Maintain readability with line breaks */ .button, .btn, [type="button"], [type="submit"] { padding: 10px 20px; }

8. Best Practices & Tips

DO's ‚úÖ

  • Use classes for styling
  • Keep specificity low and consistent
  • Use semantic selectors when possible
  • Leverage attribute selectors for state
  • Use :nth-child() for patterns
  • Group related selectors
  • Use pseudo-elements for decoration

DON'Ts ‚ùå

  • Avoid IDs for styling (use for JS only)
  • Don't over-qualify selectors (div.button ‚Üí .button)
  • Avoid deep nesting (div ul li a span)
  • Don't rely on tag names when classes exist
  • Avoid universal selector in large-scale CSS
  • Don't forget browser compatibility for newer selectors

Good vs Bad Selector Examples

/* ‚ùå BAD */ #content div.sidebar ul li a.active { color: red; /* Too specific! */ } div.button { padding: 10px; /* Unnecessarily qualified */ } /* ‚úÖ GOOD */ .nav-link-active { color: red; /* Simple, specific enough */ } .button { padding: 10px; /* Clean, reusable */ } /* ‚ùå BAD */ div div div p { color: blue; /* Fragile, depends on structure */ } /* ‚úÖ GOOD */ .content-text { color: blue; /* Clear intent, maintainable */ }
